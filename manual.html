1. 安装与启动

Windows 双击 start-flow-studio.cmd。脚本会调用 PowerShell 检查 Node.js/npm、安装 Playwright、启动本地服务并打开默认浏览器。

npm install
npm run serve
# 打开 http://127.0.0.1:8765

Git Bash、WSL、macOS 或 Linux 使用 ./start-flow-studio.sh。服务默认仅监听本机。

2. 多开浏览器与 CDP

“多开浏览器”节点先检测 CDP 端口。端口已存在时直接连接；不存在时在 Windows 启动 Edge，并使用独立用户数据目录。

& "${env:ProgramFiles(x86)}\Microsoft\Edge\Application\msedge.exe" `
  --remote-debugging-port=9222 `
  --user-data-dir="$env:TEMP\vidu-edge-profile" `
  "https://www.vidu.com/zh/create/character2video"
同一个端口和用户数据目录对应一个浏览器实例。需要多开时使用不同端口和不同目录。

3. 步骤编排

左侧选择操作，中间调整顺序，右侧只填写参数。每个步骤右上角的小眼睛控制是否生成该步骤。高级设置才允许覆盖整个代码块。

操作作用
上移/下移调整同一层级中的执行顺序。
复制复制步骤及其嵌套子步骤，并生成新 ID。
删除删除当前步骤；条件、循环或任务的子步骤会一起删除。
眼睛临时停止生成该步骤，不删除参数。
结束标志选中后插入的新步骤位于整个代码块之后。

4. 元素操作

元素节点把“如何定位”和“执行什么动作”分开。优先使用稳定的角色、标签、placeholder 或 test id，最后再使用易变化的 class/CSS。

  • 定位方式：CSS、文本、角色、placeholder、test id、现有 Locator、Locator 表达式。
  • 动作：定位、点击、等待、填写、上传、键盘输入、读取状态/值、截图。
  • nth()：多个匹配元素中选择指定序号，索引从 0 开始。
  • 键盘：普通文本使用 page.keyboard.type()；Enter 等按键使用 page.keyboard.press()。

5. 条件、循环与等待

条件节点可检测文件、变量或表达式。选中条件节点后插入的步骤成为缩进子步骤；选择“否则”分支可插入备用路径。

循环节点同样拥有缩进内容和结束标志。等待节点支持固定毫秒数，或在最小值与最大值之间生成随机等待时间。

6. 任务复用

“按序号打包步骤”选择开始和结束步骤，将连续步骤保存为现有任务。任务节点默认收起，编辑时再展开。

外部 `.mjs` 也可以作为模块任务导入。结构化任务适合继续编辑参数；模块任务适合复用已经稳定的完整脚本片段。

7. 撤回、前进与磁盘缓存

流程变更会经过短暂防抖后写入 .flow-cache/snapshots/。浏览器只保留当前流程，不把全部快照常驻内存。

  • Ctrl+Z：撤回。
  • Ctrl+Shift+Z：前进。
  • 设置页面：配置 2 到 500 个快照上限。
  • 撤回后再编辑：放弃原来的前进分支，形成新的历史分支。
  • 清空历史：删除磁盘快照，但保留当前流程并将其设为新起点。

8. 导入与导出

JSON 保存完整结构化步骤。由 PFS 生成的 MJS 内含流程元数据，可以无损恢复；普通手写 MJS 会按可识别语句拆分，无法解释的代码保留为自定义或模板代码节点。

导入仅解析文件文本，不会执行脚本。运行外部文件时，服务先复制到 runtime/scripts/，确保统一使用 IDE 目录中的 Playwright 依赖。

9. 运行、终止与日志

“运行当前脚本”把实时代码写入运行目录后启动独立 Node 子进程。页面每 600ms 读取一次状态和输出。

“终止运行”先发送中断信号；若进程未退出，Windows 会通过 taskkill /T /F 终止整个进程树。该功能只控制由当前 IDE 服务启动并记录的任务。

script-runner.mjs 在顶层模块执行完成后主动退出，因此遗留计时器或连接句柄不会让脚本无限挂起。

10. Vidu 工作流建议

  1. 先使用多开浏览器节点连接已登录的 Edge。
  2. 用角色、可见文本或 placeholder 定位“主体”和创建主体弹窗。
  3. 隐藏文件框稳定存在时直接使用 setInputFiles；动态创建时监听 filechooser。
  4. 每个页面切换使用可见元素等待，不只依赖固定时间。
  5. 最终创作按钮保留为停用步骤，人工检查后再启用。

11. 故障排查

现象检查
找不到页面确认 Edge 由远程调试端口启动，且 URL 与页面筛选条件一致。
Locator 超时检查元素是否在弹窗、iframe 或延迟渲染区域；先打印匹配数量。
上传无反应确认文件绝对路径存在、input 接受的类型正确，并等待预览或上传状态。
下一步不可用检查必填姓名、标签和图片处理状态，不要只增加固定等待。
终止后仍有浏览器CDP 模式通常复用独立浏览器实例；终止脚本不等于关闭该浏览器。
历史按钮不可用确认服务可访问，且 `.flow-cache` 目录可写。

12. 安全与许可

  • 只运行可信 `.mjs`，脚本具有当前本地用户的权限。
  • 不要在流程或仓库中保存 Cookie、Token、密码和私钥。
  • 本地服务未设计公网鉴权，不要直接暴露到互联网。
